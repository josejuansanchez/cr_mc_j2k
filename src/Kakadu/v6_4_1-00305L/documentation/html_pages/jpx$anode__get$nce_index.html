jpx_metanode::get_sequence_index

kdu_long get_sequence_index( )

[Declared in "../apps/compressed_io/jpx.h"]

Go to class description.

Synopsis

This function returns an index which identifies the point at which the current metanode was added to the hierarchy by parsing, copying or direct use of one of the add... functions, such that metanodes added later should have larger indices. This is useful if you need a means of managing a stable ordering system for metanodes. In particular, descendants of a given metanode will have monotonically increasing indices.

Although indices should be unique and should have the ordering properties mentioned above, it would be best not to rely upon this fact to the extent that your code might fail, because the internal mechanism might only use a 32-bit integer to represent sequence numbers, leading to a slim possibility of wrap-around.
