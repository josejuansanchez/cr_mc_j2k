kdu_servex::trim_resources

void trim_resources( )

Java: public native void Trim_resources( )

[Declared in "../apps/kdu_server/kdu_servex.h"]

Go to class description.

Synopsis

The application should call this function whenever the number of threads which might be using this object decreases. This is because this the number of internal kdu_event objects created to facilitate efficient implementation of the lock_codestreams and release_codestreams functions may grow until it reaches the minimum of the number of threads concurrently trying to lock codestreams and the number of codestreams in the source. When these synchronization objects are not in use, they are kept on a free list. In an application where there are a large number of source processing threads and a large number of sources, the number of synchronization objects in the system may grow to reach the product of the number of source threads and sources (unlikely, but conceivable). To avoid this happening, and ensure that the number of synchronization objects does not grow beyond the total number of source threads, each source thread should call the present function when it stops using the source (typically when it is closing a client connection).
